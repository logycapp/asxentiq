Button Guide

Documentacion visual de los botones de Bootstrap adaptada al lenguaje del dashboard de Asxentiq. Aqui reunimos estados, tamanos, variantes y patrones recomendados para mantener consistencia en toda la interfaz.

touch_app Bootstrap Buttons

1. Uso basico

La base de Bootstrap

.btn

Bootstrap usa la clase .btn como base. A partir de ella se construyen las variantes y estados que usamos en la aplicacion.

Primary action Recommended
Secondary action Complementary
Neutral action Low emphasis

2. Variantes

Intencion y prioridad

Intent

Estas son las variantes de uso mas comun en Bootstrap para comunicar intencion y prioridad.

3. Outline y link

Acciones secundarias

Subtle

Las variantes outline y link funcionan bien para acciones secundarias o interfaces con poco ruido visual.

4. Tamanos

Escala visual

Scale

Usa .btn-sm para acciones compactas y .btn-lg cuando quieras dar mas presencia.

5. Estados

Comportamiento

Behavior

El estado activo y el disabled ayudan a comunicar disponibilidad, foco y seleccion dentro del flujo.

6. Con iconos

Botones con icono

Icon button

La combinacion de texto e icono acelera el escaneo visual, especialmente en paneles densos y tablas.

7. Button group

Controles segmentados

Segmented

Muy util para filtros de periodo, vistas de analitica o estados donde solo una opcion debe quedar activa.

8. Reglas recomendadas

Buenas practicas

Best practices
Un solo boton primary por bloque para evitar competencia entre acciones.
Outline para acciones secundarias, enlaces o controles no destructivos.
Danger solo para borrar, cerrar o acciones reversibles con riesgo.
Conservar el mismo patron de colores que ya usa el dashboard.
